## Contexto de la Sesión:
Se inicia la **Fase 1.5 (Ecosistema Ravyn)** enfocada en la Landing Page y el Showcase de módulos. El objetivo es proporcionar una experiencia de preventa fluida donde los usuarios puedan previsualizar los módulos con diferentes temas.

## Logros de la Sesión:

### 1. Corrección Crítica: Showcase de Módulos (Bug del "Carga Infinita")
- **Problema:** El overlay de "Aplicando tema" se quedaba bloqueado indefinidamente al cambiar de tema en el Showcase.
- **Causa Raíz:** 
    - Uso de `srcDoc` en el `iframe` de previsualización que provocaba un ciclo de re-renderizado infinito cada vez que el estado de carga del padre cambiaba.
    - Falta de un mecanismo de "timeout" de seguridad para liberar el estado de carga si el evento `onload` del CSS fallaba o se cancelaba.
- **Solución Implementada en `ModulePreviewFrame.tsx`:**
    - **Inyección Manual:** Se eliminó `srcDoc` y se implementó `doc.write()` manual dentro de un `useEffect` con persistencia mediante `useRef` para inicializar el `iframe` una sola vez.
    - **Tracking de Estado Robusto:** Se añadió `currentThemeRef` para evitar recargas innecesarias si el tema seleccionado ya está activo.
    - **Seguridad:** Implementación de un `setTimeout` de 3 segundos que garantiza que el overlay desaparezca incluso si hay errores de red o el navegador no dispara el evento de carga del estilo.
    - **IDs Dinámicos:** Los links de temas ahora usan marcas de tiempo para asegurar que el navegador siempre aplique la versión más reciente del CSS durante la previsualización.
